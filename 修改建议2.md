为了达成 **1000步** 的目标，并突破目前 **400步** 的瓶颈，我们需要一套完整的“冲刺方案”。目前你的模型卡在 400 步且熵减到了 0.5，说明它正处于“黎明前的黑暗”。

以下是针对 **1000步目标** 重新设计的优化路线图：

### 一、 核心逻辑调整：建立“阶梯式”引导

模型现在“看不到”500步后的世界，因此它觉得活到400步已经到头了。我们必须用**面包屑奖励**把它引向1000步。

#### 1. 细化里程碑奖励 (Milestone Rewards)
不要只设 200/500/800，要在模型“没见过”的领域加密奖励点：
```python
# Preprocessor.py
milestone_reward = 0.0
# 现有瓶颈突破奖
if self.step_no == 400: milestone_reward = 2.0
if self.step_no == 450: milestone_reward = 3.0
# 核心关口奖：双怪生存
if self.step_no == 500: milestone_reward = 10.0 # 重奖，这是质变点
# 中后期过渡奖
if self.step_no == 600: milestone_reward = 5.0
# 核心关口奖：怪物加速生存
if self.step_no == 700: milestone_reward = 15.0 # 极高奖，因为难度极大
if self.step_no == 850: milestone_reward = 10.0
# 终点奖
if self.step_no == 1000: milestone_reward = 50.0 # 终极目标
```

#### 2. 增加“空间预留”奖励
鲁班死在 400 步通常是因为在单怪阶段太“安逸”，被逼到了墙角，导致无法处理 500 步刷出的第二只怪。
*   **逻辑**：在 400-500 步之间，给**地图中心区域**加奖。
*   **目的**：强迫鲁班在双怪刷新前，主动向地图中央（开阔地带）靠拢。
```python
if 400 < self.step_no < 500:
    dist_to_center = 1.0 - _norm(np.sqrt((hero_pos['x']-64)**2 + (hero_pos['z']-64)**2), 90.0)
    reward[0] += 0.05 * dist_to_center 
```

### 二、 训练参数升级：从“短跑”变“长跑”

#### 1. Batch Size：128 $\rightarrow$ 1024
*   **必须修改**：128 的 Batch Size 无法支撑 1000 步的信用分配。
*   **理由**：1024 的 Batch Size 才能让模型在一次更新中对比多局“长达 1000 步”的游戏，从而理解早期的决策如何影响后期的生死。

#### 2. 熵系数 (BETA_START)：0.008 $\rightarrow$ 0.02
*   **必须修改**：目前的 0.5 熵代表模型已经丧失好奇心。
*   **效果**：调高到 0.02 后，步数可能会暂时掉回 100-200 步，**不要慌**，这是在“破旧立新”。它会重新探索，并撞上我们设下的 450、500 步新奖励。

#### 3. 增加 PPO Epoch：建议 5 - 10
*   既然 Batch Size 变大了，每一批数据的质量更高，让模型多学几遍。

### 三、 技能逻辑：逼出“闪现”潜力

模型卡在 400 步，很大程度是因为它**还没有学会“用闪现保命”**。

*   **惩罚“带着技能死”**：
    ```python
    if done and self.step_no < 900: # 如果没到目标就死了
        if flash_ready > 0.5: # 且死的时候闪现是好的
            reward[0] -= 2.0 # 给一个重罚
    ```
*   **逻辑**：强迫模型去尝试闪现。只要它敢闪现，就有可能偶然越过怪兽，看到 500 步后的新奖励，从而进入正向循环。

---

### 四、 特征工程微调：强化“时间感”

你已经加入了 `time_to_second_monster`，但如果它没活到过 500 步，这个特征就是无效噪声。

*   **建议**：在 `time_to_second_monster` 的基础上，增加一个 `monster_danger_level` 综合特征。
*   **公式**：`danger_level = (step_no / 1000.0) * (monster_count)`。
*   **目的**：明确告诉神经网络，时间越往后，危险系数呈指数级增长，必须更加谨慎。
