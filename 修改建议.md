# 峡谷追击：鲁班七号 800 步生存与宝箱采集优化方案

## 1. 问题诊断
目前模型停留在 200 步左右的原因通常有三点：
1.  **生存奖励太低**：基础生存奖励（0.003）相对于失败惩罚（-1.2）过小，模型认为“活下去”的动力不足。
2.  **宝箱引导太弱**：宝箱奖励相对于怪物压力显得不值得冒险，模型选择了“猥琐发育”但最终被包围。
3.  **缺乏阶段性认知**：模型不清楚 500 步和 700 步会发生难度突变（双怪、加速），没有为中后期储备闪现或位移空间。

## 2. Preprocessor.py：奖励函数（Reward Shaping）深度优化

建议将奖励值放大，并引入**课程学习引导**。

### 2.1 修改基础奖励数值
在 `feature_process` 方法中修改：

```python
# --- 修改前 ---
# survival_reward = 0.003
# treasure_score_reward = 0.16 * treasure_gain

# --- 修改建议 ---
# 1. 提高基础生存奖励，让活着的“每一步”都更有价值
survival_reward = 0.01 

# 2. 大幅强化宝箱采集奖励（捡到一个宝箱相当于活100步）
treasure_score_reward = 1.0 * treasure_gain 

# 3. 优化宝箱接近奖励（Dense Reward），去掉过于严格的 safe_gate
# 让它在 20 步以外就能感受到宝箱的吸引力
treasure_approach_reward = 0.15 * treasure_approach_gain 

# 4. 引入阶段突破奖励（里程碑）
milestone_reward = 0.0
if self.step_no == 200: milestone_reward = 1.0  # 突破现有瓶颈奖励
if self.step_no == 500: milestone_reward = 2.0  # 熬到双怪出现奖励
if self.step_no == 800: milestone_reward = 5.0  # 达成目标奖励
```

### 2.2 优化闪现（Flash）与惩罚项
考虑到闪现 CD 只有 100 步，应该鼓励频繁使用闪现来调整位置或抢夺宝箱。

```python
# 5. 闪现逃生奖励增强
flash_escape_reward = 0.3 * flash_escape_event # 提高闪现成功拉开距离的奖励

# 6. 降低过度惩罚，防止AI“摆烂”
# 将原本过重的惩罚项乘一个折扣系数
punishment_scale = 0.4
enclosure_cost *= punishment_scale
danger_cost *= punishment_scale
dead_corner_cost *= punishment_scale
invalid_move_cost = -0.01 * invalid_move_event # 降低无效移动惩罚
```

## 3. Preprocessor.py：特征工程（Feature Engineering）增强

为了让鲁班学会处理 500/700 步的危机，需要把“时间压力”显性化。

### 3.1 增加时间预警特征
在 `summary_feature` 中增加对关键节点的倒计时：

```python
# 在 feature_process 中计算并加入 hero_main_feature
time_to_second_monster = _norm(max(0, 500 - self.step_no), 500)
time_to_speedup = _norm(max(0, 700 - self.step_no), 700)

# 更新 hero_main_feature
hero_main_feature = np.concatenate([
    boundary_clearance,
    np.array([
        step_norm,
        total_score_norm,
        time_to_second_monster, # 新增：距离双怪还有多久
        time_to_speedup,        # 新增：距离怪物加速还有多久
        high_pressure_stage,
        stuck_score,
        loop_score,
    ], dtype=np.float32)
])
```

## 4. 训练策略与参数微调

### 4.1 提高探索率（Entropy）
如果模型卡在 200 步，说明它已经认为这是“最优解”了。需要强制它探索：
*   **修改 `conf.py`**：提高 `BETA_START`（熵系数）。例如从 `0.01` 提高到 `0.02`。
*   **减慢衰减**：让探索时间变长，直到它偶然捡到宝箱并发现奖励巨大。

### 4.2 GAE 与 Gamma 设置
*   **Gamma ($\gamma$)**：建议设为 `0.99`。如果设得太小（如 0.9），模型会变得短视，只管当下不被打，不管 500 步后的双怪。
*   **Lambda ($\lambda$)**：建议设为 `0.95`，保持优势估计的平稳。

## 5. 阶段性训练建议

为了达成 800 步的目标，建议分两个阶段调整环境配置进行训练：

1.  **第一阶段：贪婪引导（捡宝箱训练）**
    *   暂时设置 `monster_interval: 1000`（推迟第二只怪出现）。
    *   目的：让模型在低压力环境下学会根据 `treasure_approach_reward` 走位捡宝箱。
2.  **第二阶段：抗压生存（800 步训练）**
    *   恢复 `monster_interval: 500` 和 `monster_speedup: 700`。
    *   此时模型已经带了“捡宝箱”的基因，开始学习如何在强压力下平衡生存与收益。
